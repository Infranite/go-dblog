# CI Evidence

`go-dblog` treats pull request CI as the release gate for backend behavior.
Local checks are intentionally short; fixture-backed service coverage runs in
GitHub Actions.

[中文](./CI.zh-CN.md)

## Required Checks

The protected `ci` job requires these checks before a pull request can merge:

| Area | Coverage |
|---|---|
| Unit and race tests | Root package plus short tests for MySQL, PostgreSQL, MongoDB, and Redis modules. |
| Fixture-backed integration | MySQL binlog fixtures for 5.6, 5.7, 8.0, and 8.4; PostgreSQL 16 logical decoding; MongoDB 7.0 oplog/change streams; Redis 7.2 AOF/PSYNC. |
| Parser robustness | Fuzz smoke targets for MySQL, PostgreSQL, MongoDB, and Redis parsers. |
| Parser performance | Benchmark smoke targets for MySQL, PostgreSQL, MongoDB, and Redis parsers. |
| Static gates | `golangci-lint`, `go vet`, and `govulncheck` for every module. |
| CI evidence | Tested matrix and benchmark history artifacts generated by `ci-report`. |

## Published Artifacts

Every CI run publishes small text artifacts with a 90-day retention period:

| Artifact | Files | Purpose |
|---|---|---|
| `benchmark-<module>` | `benchmark-<module>.txt` | Raw Go benchmark output for one parser module. |
| `ci-report` | `tested-matrix.md`, `tested-matrix.json`, `benchmarks.md`, `benchmarks.jsonl`, `ci-report.md` | Human-readable and machine-readable release evidence. |

The workflow summary also includes the same `ci-report.md` content so reviewers
can inspect the tested backend/version matrix and parser benchmark history
without opening job logs.

## Current Matrix

| Product | Runtime | CI scope |
|---|---|---|
| Common API | Go 1.25.x | Root package tests, registry tests, checkpoint tests. |
| MySQL | MySQL 5.6, 5.7, 8.0, 8.4 | Binlog fixture generation and module tests; MySQL 8.4 also runs live replication and decoder benchmark coverage. |
| PostgreSQL | PostgreSQL 16 | Logical decoding fixture generation, live SQL polling, wire replication reader, and module tests. |
| MongoDB | MongoDB 7.0 | Oplog fixture generation, live change streams, and module tests. |
| Redis | Redis 7.2 | AOF fixture generation, live PSYNC replication stream, and module tests. |

## Retention Scope

CI artifacts are intentionally small and text-only. They keep recent benchmark
history available inside the free GitHub Actions artifact retention window
without storing generated database fixtures or service logs.
