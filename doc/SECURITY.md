# Security Policy

[中文](./SECURITY.zh-CN.md)

## Supported Versions

Security fixes target the current public tag set `v0.4.0` and the default
branch. Supported version windows for later releases are documented in GitHub
Releases.

## Reporting a Vulnerability

Use GitHub private vulnerability reporting or open a private security advisory
for the repository. Do not publish exploit details in public issues before a fix
is available.

Include:

- affected module and version or commit;
- minimal input needed to reproduce the issue;
- expected impact, such as panic, resource exhaustion, unsafe flashback output,
  or incorrect event decoding.
